Agentic ETL Initial Setup Guide

Use this page for full machine setup, account connection, and environment configuration. Once these steps are complete, use the app quick-start guide inside the frontend for day-to-day usage.

One-time setup per machine
Includes Docker, Ollama, Google Drive, OneDrive, and .env setup

Before you begin

  • You need local admin permissions to install Docker Desktop and Ollama.
  • You should have access to project credentials and account ownership before starting.
  • If you need to set up the tool with a new OneDrive account, you will need admin permission for one step.
  • The tool requires Docker Desktop and Ollama to be running in the background before use.

Install required tools first

If you already have Docker Desktop and Ollama installed:
Open both applications and leave them running in the background while using the tool.
Make sure you see their icons in your system taskbar (bottom right on Windows, top bar on Mac).
You may close them after you are done with the tool to save RAM/battery.
Installation Step 1

Install Docker Desktop

  1. Go to docker.com/products/docker-desktop.
  2. Download Docker Desktop for your system and run the installer.
  3. Open Docker Desktop and wait until it shows that the engine is running.
  4. You may minimize or exit out of Docker Desktop, but verify that it is running in the taskbar (bottom right on Windows, top bar on Mac).
Installation Step 2

Install Ollama and pull a model

Note: The tool will automatically pull (download) the required Ollama model based on your .env file configuration (see the .env setup section below). You may skip steps 2 and 3 if you want to let the tool handle this automatically, but it may take extra time during the first usage. If you want to have the model ready to go before starting the tool, follow step 2 to pull the model, and follow step 3 to verify Ollama is not giving any errors.
  1. Go to ollama.com/download and install Ollama.
  2. (optional) Open a terminal and pull your model.
ollama pull tinyllama:1.1b
  1. (optional) Verify Ollama is reachable locally; You can run this command on any terminal, and as long as you don't get any errors, you should be all set.
curl http://localhost:11434/api/tags

Full manual setup (new account or no .env access)

If you do not have access to the .env file or want to set up this tool on a new account, follow this guide exactly. Docker and Ollama should already be installed from the section above.

Create and fill the .env file

  1. Make a copy of the file ".env.example" and rename it to .env in the project root.
  2. You may open it with your preferred text editor, Windows Notepad works perfectly. Note that the lines starting with '#' are comments and only exist for your reference. More detailed comments are included in the example file. It should look something like this:

The following steps will guide you through how to setup and configure these entries

Step 2

Google Drive setup

This step connects the tool to Google Drive so it can automatically download files from a shared folder into the project.

  1. Get the Google service account JSON key file from your admin or team. This file is named google-service-account-key.json.
  2. Place the file inside the project's secrets/ folder. The filename must be exactly google-service-account-key.json — Docker mounts this file automatically when the tool starts.
  3. Open the Google Drive folder that contains the files you want to sync. The folder ID is the string of characters at the end of the URL after /folders/.

    For example, in the URL:
    https://drive.google.com/drive/folders/1A2B3C4D5E6F7G8H
    the folder ID is 1A2B3C4D5E6F7G8H.
  4. Open your .env file and set:
    GOOGLE_DRIVE_FOLDER_ID=your-folder-id-here
  5. Make sure the service account email address has been granted at least Viewer access to that Google Drive folder. Your admin can confirm this — without it, the sync will fail with a permissions error.

Once this is done, the tool will be able to download new and updated files from the Google Drive folder whenever you run run google to onedrive in the chat.

Step 3

Set up the local OneDrive backup folder

When the tool downloads files from Google Drive, it saves a copy to a local backup folder in addition to the project input folder. This backup folder should be a folder that the OneDrive Desktop app is already syncing to the cloud, so that files are automatically backed up to OneDrive as well.

  1. Make sure the OneDrive Desktop app is installed and signed in to your organization account on this computer.
  2. Locate the synced OneDrive folder on your computer. It is typically found at one of these paths:
    Mac: /Users/your-name/Library/CloudStorage/OneDrive-YourOrganization/GoogleSheetsBackups Windows: C:\Users\your-name\OneDrive - YourOrganization\GoogleSheetsBackups
    If the GoogleSheetsBackups folder does not exist yet, create it inside the OneDrive folder.
  3. Open your .env file and set ONEDRIVE_BACKUP_DIR to that path:
    # Mac example: ONEDRIVE_BACKUP_DIR=/Users/your-name/Library/CloudStorage/OneDrive-YourOrganization/GoogleSheetsBackups # Windows example: ONEDRIVE_BACKUP_DIR=C:\Users\your-name\OneDrive - YourOrganization\GoogleSheetsBackups
  4. Confirm that the OneDrive Desktop app shows this folder as synced (not paused). When the tool downloads a file from Google Drive, it will save it to this folder, and the OneDrive Desktop app will automatically upload it to OneDrive cloud.

After this step the full flow will be: Google Drive → project input folder + local OneDrive backup folder → OneDrive cloud.

Step 3

Create a OneDrive app registration in Azure

  1. Open entra.microsoft.com and log into the account you intend to connect the tool to.
  2. Select "App registrations" on the left side
  3. Click the "New registration" button.
  4. Name it something memorable such as "OneDrive Download for Agentic ETL"
  5. Select account type as "single tenant only - your organization".
  6. Leave redirect URI blank and click "register".
  7. Now that you are on the App registration screen, you need to copy 2 of these values into the .env file
  8. Copy the "Application (client) ID" value into the .env file as ONEDRIVE_CLIENT_ID.
  9. Copy the "Directory (tenant) ID" value into the .env file as ONEDRIVE_TENANT.
  10. Next, go to the "API Permissions" section on the left sidebar, and click the "+ Add a permission" button in the middle of the page
  11. Click "Microsoft Graph"
  12. Select Delegated permissions
  13. Search for "Files.Read.All" (captilization matters), and expand the Files box
  14. Check the "Files.Read.All" permission and click "Add permission" at the bottom
  15. What this does: It allows a remote application to access OneDrive files on behalf of the user. This way, we can download onedrive files programmatically into the tool, and then process them
  16. Now, you should see the added permission in the list
  17. Next, go to the "Authentication (Preview)" section on the left sidebar.
  18. The feature we want to use is only available on the "old experience" for whatever reason. So you need to click the blue link in the blue box to switch to that experience
  19. Once you are in the old experience, first check that the dropdown box of "Supported account types" says "single tenant - your organization", and if not then change it.
  20. Next, scroll down to the "Advanced settings" section at the bottom, and change "Allow public client flows" to "Yes", and click save at the bottom. This will allow the application to properly connect
  21. Still in the Authentication old view, click the "+ Add a platform" button, and select "Mobile and Desktop Applications". This, along with the previous steps, will allow the application to function properly.
  22. In order to allow the application to log in to the Microsoft account API for you, you need to select the third option, which end in "MSAL only". Then click configure at the bottom
  23. Now, the OneDrive account should be all set up and authenticated! Next, log into the OneDrive account, and decide which folder you would like this tool to pull from. Note that it pulls all Excel and CSV files in all subfolders as well. It is recommended to use the same folder that was set up in the Google Drive section, so that files may transfer from [Google Drive -> One Drive -> Local project folders] seamlessly. Place the name of the folder you want to use in the .env file's ONEDRIVE_REMOTE_FOLDER variable, as follows:
    24. ONEDRIVE_REMOTE_FOLDER=YourFolderName

Start the Tool

Step 5

Before starting the services, double check that the .env file has been properly filled in, and that it has been saved (Ctrl + S) and closed.

By this point, all fields should be populated with the correct values. Leave the local paths as they are unless you have a specific reason to change them.

Step 4

Start services

  1. Keep Docker and Ollama running in the background.
  2. In the project root, there will be 2 buttons: start_my_app.bat and stop_my_app.bat. If they do not currently exist, you will need to instead right click in the project root, click "Open with Terminal" (PowerShell also works), and copy/paste the following command:
docker compose -f config/docker-compose.yml --env-file config/.env up --build -d web api
  • The app will take a few moments to start. When the messages at the bottom show check boxes and the terminal opens a new line, the app is ready to go!
    1. Open http://localhost:8080 in your browser.
    You are almost there! The last step is to authenticate the OneDrive account, then you can explore and use the tool. Note that there are 2 clickablebuttons at the top: "Project Quick Start Guide" and "Detailed Specifications". They will open separate web pages with different information. The quick start guide has instructions for using the tool day-to-day after setup, and the detailed specifications has in-depth technical details on how the tool works on the backend, along with diagrams.
    Step 5
    1. In the web interface chat, run authenticate onedrive.
    2. It will prompt you to go to microsoft.com/devicelogin and enter the code shown by the app.
    3. Sign in to the Microsoft account that owns the OneDrive destination.
    4. At this step, there is a high chance that Microsoft will ask for Admin approval. If this happens, contact your system administrator.
    5. Return to chat and run complete onedrive auth.
    Now, we have successfuly authenticated the OneDrive App Registration, and the tool can successfully interact with the OneDrive account and specified folder.
    Step 6

    Open the quick start guide and start using the tool!

    Shutting down the app

    1. When you are done using the tool, run the stop_program.bat file if it exists. If not, follow steps 2 and 3:
    2. Open a new terminal or reuse the same terminal from earlier steps.
    3. Run the following command to stop all running containers and turn off the tool:
    docker compose -f config/docker-compose.yml --env-file config/.env down
    1. Navigate to your system taskbar and close Docker Desktop to free up RAM.
    1. Close Ollama from the taskbar/system tray as well.

    Quick troubleshooting

    • If localhost:8080 does not load, confirm Docker containers are running.
    • If analyze fails, confirm Ollama is running and model name matches OLLAMA_MODEL_NAME.
    • If OneDrive auth fails, re-run authenticate onedrive and complete onedrive auth.
    • If sync shows no files unexpectedly, verify GOOGLE_DRIVE_FOLDER_ID and access rights.
    • If needed, use cleanup in chat and retry sync/process flow.
