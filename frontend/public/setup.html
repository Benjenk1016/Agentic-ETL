Agentic ETL Command Guide

This guide walks you through using the main commands in Agentic ETL, with step-by-step instructions and screenshots for each step. Make sure authentication and configuration are complete based on the guide "Agentic_ETL_Setup_Guide.html" in the consumer bundle folder before trying to access OneDrive or Google Drive features.

Ways to Add Files

Option 1: Upload via Chat

  1. Type input in the chat.
  2. Select your file (Excel or CSV) when prompted.
  3. Wait for upload confirmation.

Option 2: Place in Input Folder

  1. Open the data/input folder in your project.
  2. Copy or drag files directly into this folder.

Option 3: Sync from Cloud

  1. Use run file sync or download from onedrive in chat.
  2. Wait for files to appear in the input folder.

Using the tool (help)

Type help in the chat to see a list of available commands and their descriptions.

Available commands (type the command or its number): === MAIN WORKFLOWS === 1. run file processing - Detect changes in input folder & prepare files for LLM analysis 2. run file sync - Full sync: Google Drive -> OneDrive -> project input. Requires authentication tokens for both 3. process responses - Process pending LLM response JSON files in data/responses 4. input - Upload a file into project input (or place files manually in data/input) === TOOL SETUP === 5. authenticate onedrive - Start/continue browser-based OneDrive auth 6. complete onedrive auth - Explicitly complete auth after browser sign-in. Saves a token to data/state, which is used for the onedrive sync === INDIVIDUAL COMMANDS FOR TESTING === 7. analyze - AI analysis: Upload file + config, analyze with Ollama 8. download from onedrive - OneDrive download to project input (requires auth) 9. run google to onedrive - Sync files Google Drive -> OneDrive 10. cleanup - Reset all generated state (hashes, outputs, archive, etc.) except files in input folder. For testing purposes.

We know that the tool provides a range of commands and complicated workflows, but we'll walk you through them step by step. Ideally, you only need the first 3 commands. We left the other commands enabled in case they help if any errors ever occur or some individual commands are needed. We found that separating the workflows into the primary 3 commands was the most straightforward approach.

Note that either the command name or the corresponding number can be used to invoke the command. We left the option up to the user's preference.

We also know the output gets messy and long at times, and we plan to address it after the main functionality is complete.

1. Process Files (run file processing)

  1. Type run file processing in the chat and press Enter.
  2. Wait for the process to finish. Review results and follow any prompts for next steps.
run file processing

This command is the core of the ETL process. Once your files are in the project, you’ll use this workflow to process them for analysis.

  1. Detecting Changes:
    The tool first scans the input folder for any new or modified files since your last run. It compares these files to archived versions and uses hashes to quickly spot changes. If no changed files are made, the command will exit.
  2. Selecting a Config File:
    Next, the tool checks for a configuration file in data/config.
    – If no config is found, you’ll be prompted to upload one.
    – If one config is found, it’s used automatically.
    – If multiple configs are present, you’ll be asked to choose which to use.
  3. Processing Files:
    With the config selected, the tool processes each changed input file one at a time. You’ll see how many files need processing and the current progress. For each file, you’ll be asked if you want to run AI analysis, with several options available:
    • Cancel Remaining Queue: Stop the command at any time.
    • Skip: Skip a file (e.g., if it’s corrupted or not needed).
    • Auto-run analysis on all input files: Automatically process all changed files in sequence—useful for batch jobs. You may start this at any point in the batch in the workflow, but there is currently no way to stop it once started.
    • Yes: Start processing the current file. You’ll see a summary of changes, preview/edit the prompt, and send it to the local LLM for analysis.
    Note: The AI “warmup” checks that the local LLM is working before main processing, improving performance and confirming the model in use. LLM responses are saved for later steps.
  4. Finishing Up:
    After all files are processed, you’ll be prompted to continue to the “Process Responses” workflow. This is a separate step, so you can pause and return later if needed.

2. Run File Sync (run file sync)

Copies files from Google Drive to OneDrive, and from OneDrive into the data/input folder. Requires authentication for both drives (so make sure that the "authenticate one drive" command has been run before trying this, otherwise you will see an error). Note that in both cases, folders are checked recursively (meaning all files in subfolders, and subfolders of other subfolders, are checked). However, the folder structure is not maintained across the copying in either case, and all files get copied to the main directory of the next location right now (Google drive files in subfolders -> OneDrive main folder, and OneDrive files in subfolders -> data/input). The folders for both can be configured in the .env file, and for more details on that, see the guide called "Agentic_ETL_Setup_Guide.html" in the consumer-bunder folder. If you update the .env file, make sure to save it and then stop and start the app again.

  1. Type run file sync in the chat and press Enter.
  2. Wait for the sync to complete. Success or error messages will appear in chat.
run file sync

3. Process Responses (process responses)

This is the second of the file processing, and is where the LLM responses actually get used to update the Config File that was analyzed. The command processes pending all previously-saved LLM response JSON files in data/responses. The options follow a very similar structure as the "run file processing" command, and are as follows:

  • "continue": This is essentially a way to say "I will handle this one manually, archive it and go to the next response". It moves the response to the data/responses/archive folder, and in the future this can be swapped to automatic deleting once the process is robust enough.
  • "Skip": This option allows the user to skip processing a particular response. The response file will not be touched, and may be processed later in a future run of the same command.
  • "Cancel Remaining Queue": Escapes the current workflow and does not delete/archive any remaining responses.
  • "Automatically Handle": This option, which is not fully implemented as of April 6th, will allow the system to automatically update the config file based on the LLM responses. The idea is: It will show you the LLM response, propose edits and give you a preview, and allow you to accept or deny them.
    1. Type process responses in the chat and press Enter.
    2. Follow the prompts to handle or archive responses as needed.
    process responses

    Note that this command is still in development and may not be fully functional. The intention is to provide a seamless way to review and integrate LLM-generated insights into your configuration files.

    4. Upload a Single File (input)

    Edit this description: Uploads a single file (Excel or CSV) to the project input folder. This command only uploads files; you must run run file processing to process them. There are other options to get files ready to be processed (as explained at the top of this page).

    1. Type input in the chat and select your file.
    2. Wait for the upload to complete.
    input
    Note that the following 2 commands are related to OneDrive integration/authentication, and need ran once as part of the tool setup process outlined in consumer-bunder/Agentic_ETL_Setup_guide.html

    5. Authenticate OneDrive (authenticate onedrive)

    Edit this description: Starts or continues browser-based OneDrive authentication.

    1. Type authenticate onedrive in the chat and press Enter.
    2. Follow the browser prompts to sign in and complete authentication.
    authenticate onedrive

    6. Complete OneDrive Auth (complete onedrive auth)

    Edit this description: Completes OneDrive authentication after browser sign-in. Saves a token to data/state for use in OneDrive sync.

    1. Type complete onedrive auth in the chat and press Enter.
    2. Wait for confirmation that authentication is complete.
    complete onedrive auth
    Note that the following commands are for testing individual functionalities just in case the full workflow is not working as expected for whatever reason. "analyze" is built-in to the "run file processing" command, and this is more of an individualized version of it. We are leaving it in for now in case the full automation fails, or if the user just wants to run a single specific file in the case of some corruption/error somewhere. It has not been as thoroughly tested as the main workflow commands, so be weary that errors may exist. Also, if "run file sync" fails, the individual commands may be helpful for debugging which part is causing issues. "Cleanup" is useful for testing, but it deletes ALL output data. Depending on feedback, we may implement a lighter cleanup command that clears up the archives to save on storage in the long term.

    7. Analyze a File (analyze)

    Runs AI analysis on a single file and config. Useful for quick, individual file testing. For many files, use run file processing instead.

    1. Type analyze in the chat and press Enter.
    2. Select your input file when prompted.
    3. Select your config file when prompted.
    4. Wait for the analysis to complete and review the results in chat.
    analyze

    8. Download from OneDrive (download from onedrive)

    Downloads files from OneDrive to the project input folder (requires authentication).

    1. Type download from onedrive in the chat and press Enter.
    2. Wait for the download to complete. Success or error messages will appear in chat.
    download from onedrive

    9. Sync Google Drive to OneDrive (run google to onedrive)

    Syncs files from Google Drive to OneDrive (requires authentication for both).

    1. Type run google to onedrive in the chat and press Enter.
    2. Wait for the sync to complete. Success or error messages will appear in chat.
    run google to onedrive

    10. Cleanup Project State (cleanup)

    Resets all generated state (hashes, outputs, archive, etc.) except files in the input folder. For testing purposes.

    1. Type cleanup in the chat and press Enter.
    2. Wait for the cleanup to complete. Success or error messages will appear in chat.
    cleanup

    When you are finished using the tool

    1. Go back to the consumer bundle folder.
    2. Find stop_my_app.bat.
    3. Double click stop_my_app.bat to stop the program.
    4. You may also close out of Docker and Ollama to save on RAM and CPU usage if you are not planning to use the tool again soon. If you need help with this step, see the section at the bottom of Agentic_ETL_Setup_Guide.html in the consumer-bundle folder.
