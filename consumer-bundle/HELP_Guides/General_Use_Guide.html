Agentic ETL General Use

This guide is for a new user using either the live website or a locally run version of the site.

For first-time users
Local use: You will need Docker Desktop and Ollama installed, and both need to be open in the background before starting the app.

Things to check

  • Supported browser such as Chrome, Edge, or Safari.
  • The Excel or CSV files you want to upload, if your workflow includes file uploads.
  • You have completed the Initial Setup Guide.

Local setup

Step 1

Make sure Docker Desktop and Ollama are ready

  1. Open Docker Desktop and make sure it is running.
  2. Open Ollama and leave it running in the background.

Step 2

Start the local app

  1. If NO consumer-bundle folder exists, open your project folder and run docker compose -f config/docker-compose.yml --env-file config/.env up in the terminal.
  2. Find the consumer bundle folder. And double-click the start_my_app.bat file.

Step 3

Open the local website

  1. The start_my_app.bat file will open your browser automatically.
  2. This may take a few seconds while the system finishes starting.
  3. If the browser does not open, type this address into your browser:
http://localhost:8080
  1. Wait for the page titled Agentic ETL Chat to load.
  2. If the page looks incomplete, refresh the browser.

Step 4

Using the App

  1. Type help to view the available commands.
  2. Press Enter to see the list of available commands.
  3. Type the commands you'd like to execute and press Enter.
  4. If a command is not working, inform your admin if applicable.

Step 5

Done using the App?

  1. Open the consumer bundle folder.
  2. Locate stop_my_app.bat.
  3. Double click stop_my_app.bat to stop the program.
  4. Wait for the prompt window to say its ok to close the window or press any key to continue.

Most common commands

help : Display available commands === MAIN WORKFLOWS === 1. run file processing - Detect changes in input folder & prepare files for LLM analysis. 2. run file sync - Google Drive -> OneDrive -> project input. 3. process responses - Process pending LLM responses in data/responses. 4. input - Upload a file into project input (or place files manually in data/input). === TOOL SETUP === 5. authenticate onedrive - Start/continue browser-based OneDrive auth 6. complete onedrive auth - Explicitly complete auth after browser sign-in.

If you are unsure which commands to run, the ideal workflow is as follows :

  1. file sync : to get all your files from Google Drive and OneDrive into the project input folder.
  2. input : to upload a file into the project input folder if not on a cloud service (local only).
  3. run file processing : to detect changes in the input folder and use AI to process the files. then update config

When to contact your admin or project contact

  • You see an error message during a sync or upload.
  • You are not sure which file should be uploaded first.
  • You are missing access to the correct Google Drive or OneDrive location.
  • The local site does not open after starting the app.
  • Docker Desktop or Ollama is not running correctly.
